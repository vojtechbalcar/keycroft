# TypingWithAllTen Project Overview

Date: 2026-03-31
Status: distilled from reviewed idea bank and user selections

## 1. Product Summary

TypingWithAllTen is a desktop-first typing web app that should feel more like a beautiful, cozy pseudo-game than a traditional typing study website.

The core positioning is:

- a more beautiful and more motivating alternative for people who are bored of classic typing websites
- a product that makes users want to "play for a few minutes" in their spare time
- a typing experience that helps users get meaningfully better without feeling like homework

The app should not sell "perfect mastery" as the main promise. The main promise is simpler and stronger:

- get better at typing
- feel your progress
- enjoy coming back

## 2. Product Direction

The strongest direction is a cozy mastery world built around a village metaphor.

The village should begin quiet, sparse, and slightly empty, not destroyed in a dramatic way. The emotional goal is not "save a world on fire." The goal is "bring life back to a place that depends on you."

The user should feel l
ess like a visitor and more like the creator or caretaker of the world. The village should exist because they are practicing. Without the user, it stays dormant.

This should create a strong return feeling:

- users miss the place when they stop
- users want to see what changed
- users feel that typing gives life to something personal

## 3. Audience and Tone

The audience should stay broad, but the product can skew naturally toward adults and older learners who care about quality, polish, and feeling progress.

Current tone targets:

- cozy
- premium
- slightly magical
- grounded in reality
- calm rather than hyperactive

The closest reference is the emotional lane of Stardew Valley:
2
- warm
- lived-in
- satisfying to restore
- lightly magical
- never too loud

The product should still feel serious underneath, but not like a formal school program.

The brand can also benefit from a quiet founder-made feeling rather than a faceless corporate tone. A subtle personal layer like "I made this because typing websites felt dead" can help the product spread naturally without changing the interface direction.

## 4. Core Product Promise

The app should help users:

- type faster
- type more accurately
- build correct habits
- feel motivated to improve
- come back because the process feels rewarding

The product should not overemphasize status, self-optimization, or "become elite." Those ideas can exist in the background, but the foreground should be:

- progress
- flow
- visible improvement
- satisfaction

## 5. Core Loop

The core loop should be:

1. Enter your village.
2. See one clear next action.
3. Do a short typing session.
4. Improve a real skill metric.
5. Trigger immediate visible change in the village.
6. Unlock the next meaningful choice.
7. Want to come back later because the place and your skill are both growing.

Important rule:

- the app should reward real improvement, not mindless repetition

## 6. Progression Model

Progression should feel more or less linear across the main journey so the app does not give huge bursts early and then stall later.

The village should start from one central point and slowly unlock surrounding areas through:

- roads
- bridges
- rail connections
- gates
- new districts
- remote mountain or forest spaces for niche skills

The village can expand into distinct districts or sub-villages tied to skill domains. Examples:

- central square or main home area for overall progress
- garden or courtyard for early fundamentals
- workshop-adjacent district for symbols or practical typing
- riverside, forest, or mountain routes for more niche mastery
- more complex architectural detail as the user becomes advanced

Visual growth should reflect both quantity and quality:

- quantity can affect size, reach, and unlocked areas
- accuracy, consistency, and clean technique can affect detail, polish, liveliness, and refinement

## 7. World Structure

The world should not be cluttered.

The user should be able to:

- zoom out and understand progress quickly
- zoom in and appreciate details in each skill area

The world should feel alive through subtle animation:

- movement in trees and grass
- lanterns, smoke, water, wildlife, weather, or day-night shifts

The story layer should stay light. The village does not need deep lore. A small opening monologue or cutscene from one or two NPCs is enough to set the tone, then the app should largely let the user get on with it.

## 8. Onboarding and Placement

The onboarding should feel like the product is trying to fit itself to the user, not judge them.

Recommended onboarding flow:

1. Very short opening scene or NPC introduction.
2. Immediate typing test as the main starting mechanism.
3. Optional self-identification after or alongside the test, not as the main method.
4. Friendly placement into a named phase.
5. First session.
6. First visible village change within minutes.

Placement rules:

- advanced users should not be forced through beginner drills
- beginners should still receive clear guidance
- users should be able to adjust their starting point if placement feels wrong

The placement result should explain:

- where they are now
- what kind of typist they seem to be
- what the next improvement direction is

Without sounding harsh or clinical.

## 9. Learning Structure

The learning structure should be hybrid:

- guided core path for people who need direction
- optional side activities for people who want freedom

The user should never feel trapped into doing exercises they already know, but the app should still protect the integrity of the progression system.

Recommended approach:

- main path is chapter-based
- users can skip ahead through a readiness check
- direct skipping without proof should have some light friction or tradeoff
- the app can recommend side quests when it notices weak patterns

This keeps the product flexible without turning it into a shapeless sandbox.

## 10. Chapter Framework

The chapter system should be informed by current popular typing products, but translated into a warmer and more premium structure.

From the current competitor sweep:

- TypingClub emphasizes large structured courses, stars, lesson variety, adaptive lessons, stories, and posture guidance
- Typing.com emphasizes gamified curriculum, structured progression, adult and school tracks, lessons, tests, and games
- Ratatype emphasizes simpler multi-lesson courses, touch-typing fundamentals, tests, and certificates
- Monkeytype emphasizes clean test UX, custom modes, history, and minimal friction
- Nitro Type emphasizes competitive racing and friend-driven motivation

The strongest takeaway is that most products split into two groups:

- curriculum-first trainers
- test or competition-first products

TypingWithAllTen should sit between them:

- stronger learning structure than Monkeytype or Nitro Type
- much better world, UX, and emotional pull than traditional curriculum sites

Recommended chapter scaffold for V1:

1. Arrival and placement
2. Home square fundamentals
3. Home row stability
4. Left and right reach control
5. Top row integration
6. Bottom row integration
7. Capitals and punctuation in context
8. Numbers and symbols for practical typing
9. Clean flow and consistency
10. Endurance and longer passages
11. Adaptive weak-spot recovery
12. Optional specialization districts later

This framework should stay adjustable after a deeper competitor breakdown.

## 11. Quest System

Quests should exist, but not as daily chores.

The product should avoid low-value daily quest design and instead use:

- chapter quests
- capstone quests
- weak-spot side quests
- adaptive micro-quests
- weekly intentions

Important nuance:

- restoration should not be a separate special quest category because restoration is the whole product fantasy

The system should frame quests as meaningful progress arcs, not task spam.

## 12. Session Design

Sessions should skew short by default.

Recommended shape:

- quick session for a few minutes
- standard session around 10 minutes
- longer session for users who want to stay

The product should be satisfying in small bursts, but reward longer time naturally if the user wants it.

The app should aim for:

- easy entry
- one clear intention
- immediate feedback
- satisfying end-of-session reveal

Streaks can exist, but only as soft bonuses or warm reminders. They should never create the feeling that users are being punished for not showing up every day.

## 13. Skill Philosophy

The app should care about more than WPM.

Core metrics and habits:

- speed
- accuracy
- consistency
- proper finger use where technically possible
- comfort and rhythm
- confidence

The educational layer should be present but subtle. It should never feel like the app is lecturing the user. The product should feel like a game that happens to be teaching excellent habits.

## 14. Feedback and Progress

Visible progress is one of the main selling points and should be treated as a central product system.

The user should be able to see progress at several levels:

- immediate session rewards
- weekly movement
- long-term village development
- specific weak-key or weak-pattern improvement

Recommended home surface:

- village as the main visual focus
- obvious path to the next session
- one primary CTA
- a few simple key metrics like average WPM and accuracy

Deeper analytics can live below the surface.

## 15. Social Layer

The social layer should stay light at first.

Early social design goals:

- give users tasteful ways to share progress
- make the village and recap screens screenshot-worthy
- allow friend comparison later
- avoid turning the product into sweaty real-time competition

Initial friend features do not need to be ambitious. The first version can be extremely simple if the core solo experience is strong.

## 16. Monetization Posture

The free version should be fully usable.

Premium should exist for users who:

- want deeper analytics
- want richer personalization
- want more advanced content later
- want to support the product because it feels worth paying for

This is important:

- premium should feel deserved
- premium should deepen the product
- premium should not rescue a crippled free experience

Hard rules:

- no microcurrency
- no gems
- no loot boxes
- no fake energy systems
- no manipulative scarcity loops

## 17. Visual Direction

The visual system is one of the biggest commercial differentiators.

Core visual direction:

- beautiful landing page
- fast load time
- cream-forward cozy palette
- refined, warm, premium atmosphere
- strong visual identity that immediately separates the product from ugly typing sites

The practice surface should stay clean and easy while typing, but the rest of the product should feel rich and alive.

## 18. Landing Page Direction

The landing page should win on beauty, clarity, and speed.

It should communicate:

- this is not another dull typing website
- this is a more beautiful way to get better at typing
- your progress becomes visible in a living world
- you can start from your actual skill level

The landing page should sell the product through visual quality as much as through copy.

## 19. V1 Scope

Recommended V1 scope:

- desktop-first web app
- village-based world hub
- placement test
- named phases
- chapter-based guided path
- optional adaptive side quests
- short-session-first design
- strong post-session feedback
- visible world growth
- core analytics like WPM and accuracy

Things that should probably wait:

- complex multiplayer
- elaborate social systems
- desktop packaging
- multiple world themes
- deep AI story customization

## 20. AI and Narrative

AI-personalized narrative can be explored later, but it should not be a dependency for V1.

Safer near-term approach:

- simple handcrafted opening monologue
- a small set of NPC lines
- lightweight contextual narrative flavor

Possible future addition:

- small AI-assisted personalization moments based on user profile or progress

## 21. Technical Product Direction

Technical direction for V1:

- web app
- desktop-first
- performance-sensitive typing surface
- strong data design for progress, analytics, and later social systems

The architecture should preserve room for:

- future desktop app packaging
- offline support later
- friend systems later
- richer analytics later

## 23. Summary Decision

TypingWithAllTen should be:

- a beautiful desktop-first typing web app
- built around restoring and growing a village
- short-session-friendly
- strongly visual
- guided but flexible
- cozy, premium, and slightly magical
- fully usable for free
- monetized later through deserved premium depth

The strongest one-line summary at this stage is:

"A cozy village-building typing game for people who want to get better without suffering through another ugly typing website."
